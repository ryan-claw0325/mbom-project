## 一、系统架构概述

### 1.1 架构风格

采用 **前后端分离架构**（B/S 架构），前端负责页面展示和用户交互，后端负责业务逻辑和数据处理。

```
┌─────────────────────────────────────────────────────────────────┐
│                         客户端层                                  │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                    Chrome / Edge / Safari                  │  │
│  │  ┌─────────────────────────────────────────────────────┐   │  │
│  │  │                   React 18 + Vite                    │   │  │
│  │  │  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐      │   │  │
│  │  │  │EBOM导入│ │结构树 │ │批量编辑│ │工艺关联│ ...   │   │  │
│  │  │  └────────┘ └────────┘ └────────┘ └────────┘      │   │  │
│  │  └─────────────────────────────────────────────────────┘   │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                              │ HTTP/REST
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                         服务层                                   │

│  │  │                    API Gateway                        │ │   │
│  │  │                  (Express Router)                    │ │   │
│  │  └─────────────────────────────────────────────────────┘ │   │

│  │  │EBOM导入   │ │BOM管理   │ │数据校验  │ │工艺关联  │  │   │

│  │  └───────────┘ └───────────┘ └───────────┘ └───────────┘  │   │

#### 2.2.1 BOM 结构树组件

```
BomTree
├── VirtualList (虚拟滚动容器)
│   └── TreeNode[] (可见节点)
├── Node展开/收起状态管理
├── 选中状态管理
└── 右键菜单 (ContextMenu)

**Props 接口：**

```typescript
interface BomTreeProps {
  bomId: string;
  nodes: MBOMNode[];
  selectedNodeId?: string;
  expandedKeys: string[];
  onSelect: (nodeId: string) => void;
  onExpand: (keys: string[]) => void;
  onEdit: (nodeId: string) => void;

```

**性能优化策略：**

| 策略 | 实现 | 效果 |
|------|------|------|
| 虚拟滚动 | react-window FixedSizeList | 只渲染可见行，5000+ 节点流畅 |
| 节点缓存 | Map<nodeId, TreeNode> | O(1) 节点查找 |
| 展开状态 | Set<nodeId> | 高效增删查 |
| 懒加载 | 展开时加载子节点 | 减少首屏数据量 |

### 3.4 中间件设计

```typescript
// errorHandler.ts - 全局错误处理
app.use((err: Error, req: Request, res: Response, next: NextFunction) => {
  logger.error(err);
  
  if (err instanceof ValidationError) {
    return res.status(400).json({ code: 'VALIDATION_ERROR', message: err.message });
  }
  if (err instanceof NotFoundError) {
    return res.status(404).json({ code: 'NOT_FOUND', message: err.message });
  }
  if (err instanceof BusinessError) {
    return res.status(400).json({ code: 'BUSINESS_ERROR', message: err.message });
  }
  
  // 未知错误
  res.status(500).json({ code: 'INTERNAL_ERROR', message: 'Internal server error' });
});

// upload.ts - 文件上传中间件
const upload = multer({
  storage: multer.diskStorage({...}),
  limits: { fileSize: 10 * 1024 * 1024 }, // 10MB
  fileFilter: (req, file, cb) => {
    const allowedTypes = [
      'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet', // .xlsx
      'application/vnd.ms-excel', // .xls
      'application/vnd.openxmlformats-officedocument.wordprocessingml.document', // .docx
      'text/xml', // .xml
    ];
    if (allowedTypes.includes(file.mimetype)) {
      cb(null, true);
    } else {
      cb(new Error('Invalid file type'));
    }
  }
});
```
