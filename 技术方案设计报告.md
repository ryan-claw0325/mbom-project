## 一、项目概述

### 1.1 项目目标

为汽车发动机制造企业定制开发 MBOM（制造 BOM）管理模块，复现 PLM 平台标准 BOM 管理能力，包括：

1. **EBOM 多格式导入 + EBOM→MBOM 智能转换**
2. **MBOM 批量修改 + 数据校验**
3. **MBOM 与工艺规程双向关联**

### 1.2 技术栈概览

| 层级 | 技术选型 | 版本 |
|------|---------|------|
| 前端框架 | React | 18.x |
| UI 组件库 | Ant Design | 5.x |
| 构建工具 | Vite | 5.x |
| 后端框架 | Express.js | 4.x |
| 开发语言 | TypeScript | 5.x |
| 数据库 | SQLite（开发）/ PostgreSQL（生产） | 3.x / 16.x |
| 文件存储 | 本地文件系统（开发）/ MinIO（生产） | - |

### 1.3 性能目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| BOM 结构树展示 | ≤3 秒 | 5000 节点以内 |
| BOM 保存 | ≤3 秒 | 5000 节点以内 |
| 批量修改 | ≥500 条/次 | 单次操作能力 |
| EBOM 导入 | 3 种以上格式 | Excel/Word/TC XML |
| 数据校验 | 实时 + 阻断 | 可配置规则 |

### 3.2 EBOM 导入模块

#### 功能设计

| 功能点 | 说明 |
|--------|------|
| 多格式解析 | 支持 Excel (.xlsx/.xls)、Word (.docx)、Teamcenter XML |
| 文件解析库 | Apache POI (Excel)、docx4j (Word)、JAXB (XML) |
| 字段映射 | 可视化配置界面 + JSON 映射模板存储 |
| 转换引擎 | 规则配置 + 人工确认点（≤3 个） |
| 断点续传 | 大文件分片上传 |

#### 技术实现

```typescript
// 导入配置模型
interface ImportConfig {
  sourceType: 'excel' | 'word' | 'tc-xml';
  fieldMappings: FieldMapping[];
  conversionRules: ConversionRule[];
  manualConfirmPoints: string[]; // 人工干预点
}

interface FieldMapping {
  sourceField: string;      // 源字段名
  targetField: string;     // 目标字段名
  transform: 'direct' | 'toNumber' | 'toDate' | 'custom';
  defaultValue?: any;
}
```

**关键技术决策：**

| 决策点 | 方案 | 理由 |
|--------|------|------|
| 文件大小限制 | ≤ 10MB | 内存友好，适合 BOM 数据量 |
| 编码检测 | UTF-8 / GBK 自动识别 | 兼容国内企业文件 |
| 断点续传 | 前端分片 + 后端合并 | 大文件稳定性保障 |
| 解析库 | xlsx (Excel) + mammoth (Word) + xml2js (TC XML) | NPM 包，TypeScript 友好 |

### 3.3 MBOM 管理模块

#### 功能设计

| 功能点 | 说明 |
|--------|------|
| BOM 结构树 | 递归查询 + 层级缓存，支持虚拟滚动 |
| CRUD 操作 | 节点的增删改查，支持复制/粘贴 |
| 批量修改 | 按条件筛选，事务保证，数据快照，回滚能力 |
| 版本管理 | BOM 级 + Node 级双版本控制 |
| 权限控制 | 角色（工艺管理员/工艺工程师/查看者） |

#### 技术实现

```typescript
// BOM 节点模型
interface MBOMNode {
  nodeId: string;           // UUID
  bomId: string;           // 所属 BOM ID
  materialId: string;      // 物料 ID
  parentNodeId: string | null; // 父节点 ID
  level: number;           // 层级深度
  nodeSequence: number;     // 同级顺序号
  qty: number;             // 用量
  unit: string;            // 计量单位
  isKeyPart: boolean;      // 是否关键件
  processSpecId: string | null; // 关联工艺规程
  version: string;         // 节点版本
  status: 'Active' | 'PhaseOut' | 'Alternative';
}

// 批量修改请求
interface BatchModifyRequest {
  bomId: string;
  filters: QueryFilters;
  modifyType: 'replaceMaterial' | 'adjustQty' | 'updateProperty';
  modifyData: ModifyData;
  createSnapshot: boolean;  // 是否创建快照用于回滚
}
```

**关键技术决策：**

| 决策点 | 方案 | 理由 |
|--------|------|------|
| 层级表达 | parent_node_id + level | 兼顾查询效率和递归展开 |
| 锁机制 | 乐观锁（版本号） | 高并发友好，避免死锁 |
| 事务边界 | 单节点 vs 批量分开 | 批量操作长事务风险隔离 |
| 前端渲染 | 虚拟列表（react-window） | 5000+ 节点流畅渲染 |

### 3.4 数据校验模块

#### 功能设计

| 功能点 | 说明 |
|--------|------|
| 内置规则 | 6 条基础规则（R001-R006） |
| 自定义规则 | JSON 条件表达式，业务人员可配置 |
| 阻断机制 | Error 阻断提交，Warning 提示 |
| 校验报告 | PDF/Excel 导出 |

#### 规则定义

| 规则编码 | 规则名称 | 类型 | 错误级别 |
|---------|---------|------|---------|
| R001 | 物料编码非空 | Mandatory | Error |
| R002 | 数量必须为正 | Range | Error |
| R003 | 物料编码格式校验 | Format | Warning |
| R004 | 层级深度不超过 10 层 | Range | Warning |
| R005 | 父件必须存在 | Relationship | Error |
| R006 | 关键件必须关联工艺 | Relationship | Error |

#### 技术实现

```typescript
// 校验规则模型
interface CheckRule {
  ruleId: string;
  ruleCode: string;
  ruleName: string;
  ruleType: 'Mandatory' | 'Range' | 'Format' | 'Relationship' | 'Custom';
  targetField: string;
  conditionExpr: string;  // JSON 条件表达式
  errorMsg: string;
  severity: 'Error' | 'Warning';
  isActive: boolean;
}

// 校验执行器
class ValidationEngine {
  async validate(nodes: MBOMNode[], rules: CheckRule[]): Promise<ValidationResult>;
  async validateNode(node: MBOMNode, rules: CheckRule[]): Promise<ValidationResult>;
}
```

### 3.5 工艺关联模块

| 双向关联 | MBOM → 工艺规程 + 工艺规程 → MBOM |

interface NodeSpecRel {
