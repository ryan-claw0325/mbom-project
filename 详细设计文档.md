## 一、数据库设计

### 1.1 数据库选型

| 环境 | 数据库 | 版本 | 说明 |
|------|--------|------|------|
| 开发 | SQLite | 3.x | 零配置，单文件，便于开发 |

## 三、核心算法设计

### 3.1 EBOM 解析算法

```
1. 接收上传文件，根据 sourceType 选择解析器
2. Excel: 使用 xlsx 库读取为 JSON 数组
3. Word: 使用 mammoth 库转换为 HTML，再提取表格数据
4. TC XML: 使用 xml2js 解析为 JS 对象
5. 识别表头行和数据行
6. 返回预览数据和未识别字段列表
```

### 3.2 字段映射转换算法

```
1. 解析 fieldMappings 配置
2. 遍历源数据每一行
3. 对每个映射字段：
   - direct: 直接赋值
   - toNumber: 转换为数字
   - toDate: 转换为日期
   - custom: 执行自定义转换函数
4. 遇到人工确认点时暂停，等待用户确认
5. 继续执行直到完成
6. 返回转换结果和失败记录
```

### 3.3 BOM 结构树构建算法

```
输入: 扁平化的节点数组 nodes[]
输出: 树形结构

1. 创建 Map<nodeId, node> 用于 O(1) 查找
2. 遍历 nodes，构建 parentNodeId → children 的映射
3. 找出根节点（parentNodeId == null）
4. 递归构建子树:
   function buildTree(node):
     node.children = getChildren(node.id)
     for child in node.children:
       buildTree(child)
     return node
5. 按 nodeSequence 排序同层节点
```

### 3.4 批量修改算法

```
输入: bomId, filters, modifyType, modifyData

1. 根据 filters 条件查询匹配的节点 IDs
2. 创建数据快照（可选，用于回滚）
3. 开启事务
4. 分批处理（每批 100 条）:
   - 应用修改
   - 写入变更日志
5. 执行校验（如果启用）
6. 提交事务或回滚
7. 返回修改结果
```

### 3.5 校验规则引擎算法

```
输入: nodes[], rules[]

1. 遍历每条规则
2. 根据 ruleType 执行不同的校验逻辑:
   - Mandatory: 检查字段非空
   - Range: 检查数值范围
   - Format: 检查正则匹配
   - Relationship: 检查关联存在性
   - Custom: 执行 JSON 条件表达式
3. 汇总校验结果
4. 如果 stopOnError=true，遇到 Error 立即停止
5. 返回详细的校验报告
```

## 四、错误处理设计

### 4.1 错误类型定义

```typescript
// types/errors.ts
export class AppError extends Error {
  constructor(
    public code: string,
    message: string,
    public statusCode: number = 400,
    public details?: any
  ) {
    super(message);
    this.name = 'AppError';
  }
}

export class ValidationError extends AppError {
  constructor(message: string, details?: any) {
    super('VALIDATION_ERROR', message, 400, details);
    this.name = 'ValidationError';
  }
}

export class NotFoundError extends AppError {
  constructor(resource: string) {
    super('NOT_FOUND', `${resource} not found`, 404);
    this.name = 'NotFoundError';
  }
}

export class BusinessError extends AppError {
  constructor(message: string, details?: any) {
```
